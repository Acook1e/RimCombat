============================================================
Rim Combat — 上古卷轴5 近战战斗重构
============================================================

版本：0.1.0（Pre-Alpha 测试版）
适用：Skyrim SE/AE 1.6.629+
状态：系统已可玩，数值持续调优中。欢迎测试反馈。


============================================================
模组简介
============================================================

Rim Combat 是一套面向上古卷轴5的完整近战战斗重构。它的核心设计思路不是单纯叠加攻击动作，而是构建一套多层次的资源博弈系统——每一次出刀、格挡和战技释放，都是一次有代价、有回报的决策。

当前版本包含以下完整的战斗模块：


============================================================
一、武器分类与基础数值层
============================================================

Rim Combat 在原版五大武器类别（单手剑/单手斧/单手锤/双手剑/双手斧锤）的基础上，扩展到了 25 种以上的武器子类型，包括细剑、武士刀、枪矛、长柄、鞭、拳套、长棍等。

每种子类型都拥有独立的：
- 基础耐力消耗
- 基础架势伤害（破防能力）
- 基础韧性伤害（打出硬直的能力）
- 格挡强度

安装 OCF（Open Animation Replacer 配套的武器分类框架）后可以识别更细的子类型。不安装 OCF 也能正常运行，此时会退回到更粗的武器分类路径。

此外还有 50 种以上的生物种族（狼、熊、巨魔、龙、查鲁斯、矮人机械等），每种都有独立的生物耐力消耗、生物架势伤害和生物韧性伤害。生物使用爪击、撕咬、撞击等攻击时，会读取种族专属的数值，而不是套用统一的"空手"模板。


============================================================
二、攻击耐力系统
============================================================

每把武器每次挥砍都消耗耐力。消耗量由以下因素共同决定：
- 武器类型基础值（匕首 5.0 / 剑 7.0 / 战锤 14.0）
- 武器质量附加（每单位重量 +0.2）
- 攻击类型倍率：轻攻击 1.0× / 重攻击 2.5× / 盾击 0.4× / 猛击 1.5×
- 战技 Payload 覆盖（由动画注释独立控制）

耐力再生同样被重新设计：
- 全局再生倍率 5.0×
- 再生延迟 1.5 秒
- 战斗中再生倍率 1.0×
- 格挡时再生倍率 0.5×

开启"耐力为零时禁用攻击"选项后，耐力见底将无法出刀——对 NPC 同样生效。这个系统通过追踪角色攻击状态（AttackState Enum）来触发消耗，确保即使在使用 MCO 或 BFCO 框架时也能正确响应每一次挥砍。


============================================================
三、力竭系统
============================================================

当耐力降到零时，角色进入力竭状态。该状态下：

攻击侧惩罚：
- 造成的伤害降低 30%
- 造成的架势伤害降低 30%

受击侧惩罚：
- 受到的伤害增加 50%
- 受到的架势伤害增加 50%

NPC 力竭时默认禁止攻击（防止无限制压制玩家）。玩家是否在力竭时禁止攻击可独立配置。此外还有"受击退出力竭"选项——被击中后自动解除力竭状态，耐力恢复到 20% 以上时完全退出。


============================================================
四、架势系统（破防）
============================================================

架势是本模组的核心压力资源，类似于"格挡槽"。

最大架势值由以下公式计算：
  基础架势（种族值） + 生命值 × 6% + 耐力值 × 4%

架势伤害由攻击方决定：
  武器基础架势伤害 × 攻击类型倍率

攻击类型倍率：
- 轻攻击：1.00×
- 重攻击：2.00×（主要破防手段）
- 盾击：1.20×（轻微压力）
- 猛击（格挡重击）：3.50×（近破防）

护甲值会通过 1.1 的独立因子减少承受的架势伤害。架势恢复延迟为 5 秒，每秒恢复 3%。架势条通过 TrueHUD 显示。

架势被打空时触发 PostureBreak 级别的硬直——这是整个战斗循环中最关键的破绽窗口。


============================================================
五、韧性与硬直系统
============================================================

韧性是一个"内部硬直阈值"——它决定了每次受击会导致什么级别的硬直反应。

最大韧性值由以下公式计算：
  基础韧性（种族值） + 耐力值 × 2% + 体重 × 1.2 + 护甲部位加成

护甲部位加成（轻甲/重甲分别）：
  头部 +0.35/+0.80
  躯体 +1.00/+2.50
  手部 +0.20/+0.60
  脚部 +0.20/+0.60

韧性伤害 = 武器基础韧性伤害 × 攻击类型倍率（同架势倍率体系）

硬直触发规则：
- 韧性被打空时触发硬直
- 单次受到 ≥ 最大韧性 70% 的伤害时触发补偿硬直（不要求韧性归零）
- 硬直等级由当次韧性伤害值决定：≥ 2.0 Small / ≥ 5.0 Medium / ≥ 9.0 Large
- 处决状态下触发升级硬直

特殊硬直等级（不会被普通硬直覆盖）：
- Knockdown（砸趴）：101
- Knockaway（击飞）：100
- Strikefly（挑飞）：102
- GuardBreak（破防硬直）：200 —— 由弹反或格挡耐力耗尽触发

同等级硬直再次触发时刷新恢复时间，不会重复播放动画。更高等级硬直会覆盖低等级，免疫状态的 NPC 可以抵抗特定等级以下的硬直。


============================================================
六、格挡与限时格挡
============================================================

格挡系统是本模组最成熟的模块之一：

普通格挡：
- 每把武器有独立的格挡强度（盾 48 / 巨剑 34 / 匕首 14 / 弓 6）
- 格挡强度决定伤害减免上限
- 每次格挡会消耗耐力，消耗量受格挡强度和伤害值共同决定
- 单次格挡最多消耗 50% 最大耐力

限时格挡（Timed Block）：
- 格挡开始后的 150ms 内为限时窗口
- 限时格挡成功后获得 500ms 的强化状态
- 强化期间：格挡强度 ×4.0，可免疫破防（可配置）
- 强化期间：免疫后座力和硬直

弹反（Parry，由格挡猛击触发）：
- 弹反是一种特殊的防御性动作——无攻击判定，纯防御
- 弹反成功直接清零本次伤害
- 给予攻击者 GuardBreak 级别的硬直
- 弹反优先级高于 GP 和限时格挡


============================================================
七、战技系统
============================================================

战技是分配在武器上的特殊技，消耗魔力（FP）释放，具有 Eligible/Subordinate 双模式。

当前首发 7 个战技：

■ 二连斩（三段，共 6 刀）
  武器：Slash 斩击类（剑/武士刀/巨剑/斧/长柄等），全重量级
  FP：三段合计 30
  效果：2 快刀试探 → 3 连打压制 → 1 重尾刀收尾。速度是核心优势。

■ 贯穿
  武器：Thrust 刺击类（细剑/匕首/矛/刺剑等），全重量级
  FP：14
  效果：单发蓄力突刺。架势和韧性倍率高达 2.5×，突出"穿透破防"。

■ 狩猎巨人
  武器：Heavy 重型 + Thrust 刺击（巨剑/战斧/战锤/长柄的刺击动作）
  FP：35
  效果：向前突刺，命中给予 Strikefly（挑飞）级别硬直。两段判定。

■ 狮子斩
  武器：Heavy 重型 + Slash 斩击（巨剑/战斧/战锤/长柄的斩击动作）
  FP：28
  效果：跃起后向下重劈，命中给予 Knockdown（砸趴）级别硬直。两段判定。

■ 居合 — 横斩（轻派生）
  武器：Normal 中型 + Katana 武士刀
  FP：8
  效果：收刀入鞘后快速横斩。快，均衡，FP 消耗极低。

■ 居合 — 竖劈（重派生）
  武器：Normal 中型 + Katana 武士刀
  FP：10
  效果：收刀入鞘后蓄力竖劈。基于重攻击，享受重击的架势/韧性倍率加成。

■ 真空斩
  武器：Slash 斩击类 + Normal/Heavy 重量级
  FP：26
  效果：蓄力下斩，挥出剑气。剑身命中走物理管道，剑气走法术管道独立结算。近身全吃两段伤害。

所有战技的伤害/架势/韧性/耐力均由动画 Payload 独立控制。魔力不足时进入 Subordinate 模式——不扣魔，但各项数值大幅降低。

此外支持：
- 按种族路由的特殊空手战技（人类空手/狼人/熊人/吸血鬼领主）
- PrismaUI 战技分配菜单（背包打开时右半屏显示）
- PrismaUI 战技 HUD（显示当前武器战技名称与状态）
- 战技经验成长（战技造成伤害获得经验，升级获得点数）


============================================================
八、伤害修正层
============================================================

攻击命中时的伤害倍率调整，分层处理：

第一层：原版倍率归一化
- 重攻击 ÷ 2.0（引擎自带重击 2.0× 膨胀）
- 盾击 ÷ 0.5（引擎自带盾击 0.5× 压缩）

第二层：全局攻击类型倍率
- 重攻击 × 1.8
- 盾击 × 0.20
- 猛击 × 0.35

第三层：战技 Payload 覆盖
- 通过 RimDamage.SetMult 设置最终倍率

第四层：状态修正
- 力竭状态增减伤
- 处决状态增伤


============================================================
九、设置与本地化
============================================================

所有数值通过 Data\SKSE\Plugins\RimCombat\Settings.json 配置。结构如下：
- Damage：伤害倍率
- Stamina：耐力消耗、再生、各武器/生物基础值
- Posture：架势基础值、各武器/生物架势伤害
- Poise：韧性基础值、各武器/生物韧性伤害、Stagger 阈值
- Block：格挡强度、限时格挡窗口
- WeaponArt：HUD 位置与缩放
- Exhausted / Stagger：行为开关与恢复时间

通过 SKSEMenuFramework 可在游戏内调整大部分标量设置（武器表和种族表仍在 JSON 中维护）。

本地化文本存放在 Localization.json，支持中文字段。


============================================================
前置需求
============================================================

必须安装：
- SKSE（1.6.629+）
- Address Library for SKSE Plugins
- OAR（Open Animation Replacer）

动画框架（二选一）：
- MCO 或 BFCO

推荐安装：
- PrismaUI —— 战技菜单与 HUD
- TrueHUD —— 架势条显示
- Modern Stagger Lock —— 硬直动画兼容
- OCF —— 更细的武器子类型识别
- Precision —— 精确碰撞检测
- Behavior Data Injector —— 战技动画注入


============================================================
安装方法
============================================================

1. 安装所有必须前置。
2. 安装你选择的动画框架（MCO 或 BFCO）。
3. 用 Mod 管理器安装 Rim Combat。dist/ 文件夹结构对应游戏根目录。
4. 如果使用战技，确保 OAR 配置路径
   Meshes\Actors\Character\Animations\OpenAnimationReplacer\RimCombat - WeaponArt\
   下的动画已启用。
5. 进游戏，打开 SKSEMenuFramework 设置菜单，
   确认 UseAttackStaminaSystem / UsePostureSystem / UsePoiseSystem /
   UseBlockSystem / UseWeaponArtSystem 均已启用。

如需重置所有设置为默认值，将 SettingsDefault.json 复制覆盖 Settings.json 即可。

============================================================
数值设计说明
============================================================

本模组的数值基于以下设计原则：

四种攻击类型各有明确定位：
- 轻攻击：基准线（1.0×），安全压制和连段起手
- 重攻击：高投入高回报（2.0× 架势/韧性），破防和惩罚技
- 盾击：廉价打断（0.20× 伤害 / 1.20× 架势），防御性工具
- 格挡猛击：高风险高回报（3.50× 架势/韧性），盾牌重击

武器数值按重量级分层：
- Light（匕首/爪/拳套/鞭）：低消耗、低冲击、高速度
- Normal（剑/武士刀/单手斧/锤）：均衡参考线
- Heavy（巨剑/战斧/战锤/长柄）：高消耗、高冲击、慢动作

生物数值与其游戏中的威胁等级对齐：
- 小型（狼/狐狸/螃蟹）：低威胁
- 中型（熊/剑齿虎/巨魔）：标准威胁
- 大型（巨人/猛犸/龙）：高威胁

数值的进一步调整需要实际战斗数据支撑——这正是本次 0.1.0 测试版的核心目的。


============================================================
反馈方式
============================================================

如果你在测试中遇到了以下情况，请在 Posts 或 Bugs 区留言：

- 某个武器类型明显过强/过弱
- 某个战技的伤害或 FP 消耗不合理
- 某些生物的攻击完全感觉不到威胁 / 过于致命
- 硬直触发频率或等级不符合预期
- 架势破防后的反馈不够明显

反馈时请尽量带上：
- 使用的是什么武器
- 面对的是什么敌人
- 具体的数值感受（太高/太低/刚好）
