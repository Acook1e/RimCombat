# Rim Combat

> 面向上古卷轴 5 SE/AE 的近战战斗重构原型。  
> 当前重点不是做一套“花哨动作展示”，而是把耐力、力竭、架势、格挡、处决和战技串成一条更强调风险、节奏和站位的近战循环。

Rim Combat 目前已经是可运行、可调数值、可进游戏实测的原型，但还不是完成版。它更适合用来测试战斗手感、系统联动和内容方向，而不是直接当作长期稳定发布版来理解。

如果你是想使用模组的玩家，请读这份 README。  
如果你是基于本项目继续开发、做动画联动或扩展规则的作者，请读 TODO.MD。

---

## 当前定位

- 当前阶段：可运行原型 / Pre-Alpha
- 当前重心：把核心战斗循环接通，并验证数值、状态切换、事件链路和 UI/HUD 联动
- 当前不追求：完整演出、全面 NPC 覆盖、正式发布级的稳定性与平衡

一句话概括现在的项目状态：

Rim Combat 的核心机制已经进游戏了，但“内容深度”和“最终打磨”还远没有完成。

---

## 当前已实现的内容

### 核心战斗循环

- 攻击耐力系统已经接通。普通攻击和重击会按武器类型、质量和攻击方式消耗不同耐力。
- 力竭系统已经接通。耐力过低后会进入力竭，并影响攻击权限、伤害和受击表现。
- 架势系统已经接通。近战受击会累积架势伤害，破韧后进入可处决状态。
- 格挡与限时格挡已经接通。格挡会参与减伤、耐力交换和架势博弈，限时格挡提供额外收益。
- 处决系统已经接通基础逻辑。目标进入可处决状态后，可以按距离、角度、武器和目标类型触发原型级处决。

### 战技与界面

- 战技菜单、HUD、武器分配、准备态 / 启用态 / 执行中状态已经接通。
- 战技菜单会跟随背包当前选中的武器刷新，可直接进行分配和切换。
- 战技效果开始转向动画事件 payload 驱动，已经接通耐力消耗、部分伤害和处决相关处理。
- 本地化和 UI 文本解析已经接通，HUD 和菜单不再只依赖硬编码字符串。

### 受击与特殊硬直

- 常规硬直不打算由 Rim Combat 单独重做，而是偏向与 Maxsu Poise、Modern Stagger Lock 配合。
- Rim Combat 当前已经接通对原版 TryStagger 的覆写链路，可以在受击前后修正最终传入值。
- 额外硬直等级与免疫规则已经有基础框架，支持通过动画事件传递目标设定、等级修正和免疫信息。

## 当前依赖与推荐联动

### 核心运行依赖

- SKSE
- OAR
- MCO 或 BFCO

### 推荐联动

- TrueHUD：显示架势条
- PrismaUI：战技菜单与 HUD
- Maxsu Poise：常规韧性 / 受击基础
- Modern Stagger Lock：常规硬直阈值与稳定化

如果缺失显示层依赖，部分逻辑仍可能运行，但 HUD、菜单和反馈会明显不完整。  
如果缺失 Maxsu Poise 或 Modern Stagger Lock，Rim Combat 当前设计下的常规受击体验也会明显不完整。

## 当前已知不足

下面这些不是“未来可能会做”，而是当前版本里明确还没收尾的部分：

- 游戏内设置菜单还很简陋，现阶段主要只有读取与重置，详细调参仍以 Settings.json 为主。
- 战技系统目前主要围绕玩家侧使用，NPC 战技分配和使用逻辑还没有真正完成。
- 战技成长、保存与更完整的数据驱动还没完全收尾，仍属于半成品。
- 处决系统已经能跑通原型，但位移锁定、旋转锁定、镜头、安全收尾和更多种族支持都还没完成。
- 特殊硬直现在主要是规则和接口先接通了，真正完整的动作表现、反馈和边界处理仍然不足。
- 常规硬直体验仍依赖外部模组生态，不建议把 Rim Combat 当成一套完全独立闭环的受击系统来使用。

## 当前可能存在的问题

这是基于当前代码状态做的项目审查后，比较值得留意的风险点：

- 架势系统里与 Breakable 相关的 payload 逻辑仍需要继续验证，当前实现对 true / false 的处理存在明显风险。
- Breakable 的复位路径还不统一，某些动作如果显式改写了状态，后续动作可能会继承错误的不可破防状态。
- 处决系统目前只真正接通了人类目标的基础组合，更多种族虽然有分类框架，但还没有完整接入可用动画链路。
- 执行处决时仍缺少位移 / 旋转锁定，属于当前最明显的演出与稳定性短板之一。
- 设置项里有少数配置已经接入读写，但还没有完全接进行为路径，需要继续核对配置和实际效果是否一致。

这些问题并不意味着当前版本不能用，而是意味着它更适合测试和迭代，不适合用“正式发布模组”的预期去要求它。

## 当前更适合怎样使用

如果你现在想试用 Rim Combat，比较适合把它当成下面这几类体验：

1. 测试不同武器下的耐力成本、压制节奏和风险感。
2. 测试力竭、格挡、架势、破韧和处决之间是否形成了你想要的战斗循环。
3. 测试战技菜单、HUD、准备态 / 启用态和武器分配流程是否顺畅。
4. 测试与 Maxsu Poise、Modern Stagger Lock 联动后的受击与硬直衔接是否自然。
5. 测试 Settings.json 调参后的行为变化是否符合预期。

## 面向用户的结论

Rim Combat 现在最有价值的部分，不是“它已经完成了多少内容”，而是“它已经把最关键的战斗主循环接通了”。

如果你想体验一套更强调耐力、架势、格挡、破韧、处决和战技联动的近战原型，它现在已经值得进游戏测试。  
如果你想要的是一套完成度很高、内容很全、演出很稳的正式战斗 overhaul，那它还在路上。
