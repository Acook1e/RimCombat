# Rim Combat

> 一个面向《上古卷轴 5》SE/AE 的近战战斗重构模组原型。  
> 它的目标不是单纯抬高伤害或替换动作，而是把耐力、力竭、架势、破韧、处决和战技组织成一套更强调节奏、风险与站位判断的近战循环。

Rim Combat 围绕 MCO / BFCO、OAR、TrueHUD、PrismaUI 等生态进行设计，整体思路明显受到《艾尔登法环》《只狼》《仁王》等作品启发，但最终仍然服务于 Skyrim 自身的战斗节奏、模组生态与可扩展性。

---

## 总描述

Rim Combat 想做的，不是“让战斗更花哨”，而是“让每一次进攻、防守和失误都更有代价”。

在这套设计里：

1. 攻击会真实消耗耐力，连续压制不再几乎没有成本。
2. 耐力过低会进入力竭，角色的攻防表现都会明显下滑。
3. 近战命中会累积架势伤害，持续制造破韧窗口。
4. 格挡与限时格挡不只是防住攻击，还会参与耐力交换、减伤和架势博弈。
5. 目标破韧后可以继续压制，也可以寻找站位触发基础处决。
6. 战技作为更高风险、更高收益的动作分支，最终会承担整套循环里的爆发与变化。

当前版本已经可以跑通一条完整的近战原型循环：攻击消耗耐力，力竭影响攻防，受击累积架势，破韧进入可处决状态，格挡 / 限时格挡介入耐力与架势交换，战技菜单与 HUD 也已经可以进行基础体验。

不过，它仍然是一个明显处于持续迭代中的原型版本，不适合作为“正式完成版”来理解。

## 模块描述

### 攻击耐力系统

Rim Combat 已经重做了近战攻击的耐力消耗逻辑。当前耐力消耗会综合武器类型、攻击类型和武器重量进行计算，不同武器之间的进攻成本已经不再完全一样。

这一模块已经能稳定参与实际战斗循环，也是当前最接近“可长期调数值”的部分之一。

### 力竭系统

当耐力被消耗到过低时，角色会进入力竭。力竭状态下，攻击伤害、架势伤害、受击表现以及部分攻击权限都会受到影响。

这套系统已经能稳定参与节奏控制，目前的主要工作重心更多放在平衡和体验打磨，而不是补基础逻辑。

### 架势条系统

模组新增了架势条这一资源。角色在受到攻击时会持续损失架势，架势值归零后会进入破韧状态，并进一步接通处决逻辑。

当前架势系统已经完成了最大值计算、恢复逻辑、护甲对架势伤害的影响，以及与格挡、力竭、处决之间的基础联动。

### 格挡 / 限时格挡系统

当前格挡不再只是“挡住或没挡住”的二元判断，而是会依据武器类型映射格挡强度，进一步影响减伤、耐力消耗、架势伤害减免，以及对攻击者的返架势效果。

限时格挡则在普通格挡基础上追加了更高强度的窗口与持续时间效果，目前已经能稳定参与整套循环测试，也是当前完成度最高的核心模块之一。

现阶段这部分更需要的是数值继续微调、音效特效与击中反馈补完，而不是重做底层逻辑。

### 处决系统

当目标进入破韧状态后，处决系统会根据站位、距离、目标种类和武器类型，判断是否可以触发正面处决或背刺处决。

当前这部分已经能完成基础状态切换、角度判定、触发检查和原型级联动，但仍然明显偏向机制验证阶段。它现在更像是“处决框架已通”，而不是“最终演出已经完成”。

### 战技系统

战技系统目前已经具备基础菜单、HUD、解锁、分配和切换流程，玩家可以在背包界面中查看战技信息，并把战技分配给武器使用。

但这部分仍然是当前最典型的半成品区域之一。虽然已经支持从 JSON 读取战技数据，也具备玩家等级、经验和点数等基础结构，但更完整的武器限制、伤害类型、点数消耗规则、持久化保存和 NPC 侧逻辑还没有真正收尾。

## 其他用户说明

### 当前已知限制

1. 当前大量数值仍在快速调整，现阶段更适合把 Rim Combat 当成测试版原型来体验。
2. 大多数数值配置目前仍以 Settings.json 为主，游戏内设置菜单现阶段主要提供读取与重置，不是完整的设置界面。
3. 同一个 FormID 的武器当前共享同一战技分配，不是按单个武器实例分别绑定。
4. 战技系统虽然已经接通菜单、HUD、解锁与分配流程，但部分数据仍然处于测试态，尚未达到完整正式的数据驱动状态。
5. NPC 战技相关内容目前还没有完成，因此当前体验仍然主要以玩家侧为主。
6. 处决系统已经可以触发基础判定与原型联动，但离最终演出效果和更广泛种族兼容还有明显距离。
7. 格挡系统的核心机制已经进入稳定测试阶段，但视觉反馈和与更多模块的联动接口还会继续扩展。

### 使用前说明

Rim Combat 当前依赖以下生态组件来获得完整体验：

| 组件 | 作用 |
| --- | --- |
| SKSE | 插件运行基础 |
| MCO / BFCO | 攻击动作框架 |
| OAR | 动作切换与条件分发 |
| TrueHUD | 架势条显示 |
| PrismaUI | 战技菜单与 HUD |

如果缺失其中的显示层依赖，部分核心逻辑仍然可能运行，但 HUD、菜单或整体反馈会明显不完整。

### 当前更适合怎样体验

如果你现在想试用 Rim Combat，比较值得重点关注的是：

1. 不同武器类型下的攻击耐力消耗与压制成本是否合理。
2. 0 耐力与力竭状态下的战斗节奏是否真的形成压迫感。
3. 架势削减、恢复、破韧与退出可处决状态的节奏是否自然。
4. 不同武器类型下格挡 / 限时格挡的减伤、耐力交换与返架势反馈是否合理。
5. 正面处决与背刺处决的触发稳定性，尤其是近距离站位判断是否自然。
6. 战技菜单、HUD、战技切换与武器分配流程是否顺畅。
7. 修改 Settings.json 后再读取 / 重置设置时，整体行为是否符合预期。

### 当前不建议的使用方式

1. 不建议把当前版本视为正式发布版使用。
2. 不建议在长期正式存档中深度依赖当前版本。
3. 不建议现在就基于它做最终平衡判断，因为多个系统仍在快速迭代。
