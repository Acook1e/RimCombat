# Rim Combat

> 面向上古卷轴 5 SE/AE 的近战战斗重构原型。
> 
> 当前版本的重点，不是单纯堆叠动作演出，而是先把耐力、力竭、架势、格挡、战技与特殊硬直接成一条可以反复实测、可以持续调数值、也可以继续扩展的近战循环。

> 本 README 的完成度评估不计入处决系统。
> 
> 处决模块目前因为 OAR 相关联动问题暂时搁置，所以当前版本的主要测试重心是非处决战斗循环。

---

## 总描述

- 当前阶段：可运行、可调参、可进游戏长期实测的 Pre-Alpha。
- 当前结论：已经到达“可以正式开始整体测试”的阶段，不再只是单个半成品模块堆在一起。
- 当前强项：攻击耐力、力竭、架势、格挡与限时格挡、玩家侧战技分配与 HUD 已经接通成链。
- 当前短板：战技内容量、NPC 战技逻辑、特殊硬直细则、表现层反馈仍在补完。
- 当前不建议期待：正式发布级平衡、完整 NPC 覆盖、成熟的处决演出与镜头。

如果只用一句话概括现在的项目状态：

Rim Combat 的核心近战循环已经成型，并且已经具备持续测试价值；但它目前仍然更接近“高可玩原型”，还不是“完整发布版战斗 overhaul”。

## 完成度

下面的完成度是按“当前能否测试、是否已经接进主循环、还差多少收尾”来评估，不是精确工程学百分比。

| 范围 | 当前判断 |
| --- | --- |
| 核心近战循环（不计处决） | 约 75% 到 80%，已经进入联调与数值打磨阶段 |
| 玩家侧可用性 | 约 70% 到 75%，基础 HUD、菜单、配置与主要玩法入口已经具备 |
| 表现层与内容量 | 约 45% 到 55%，玩法骨架在前，内容深度和反馈仍明显不足 |
| 当前是否适合测试 | 是，已经适合持续实机测试 |

当前最适合做的事，不再是怀疑“项目有没有进入测试阶段”，而是：

1. 继续验证数值手感。
2. 补全表现和反馈。
3. 把已经接通的系统做更完整的内容扩展。

## 其他用户说明

### 当前依赖与推荐联动

| 类型 | 说明 |
| --- | --- |
| 必需 | SKSE |
| 动画相关 | OAR、MCO 或 BFCO |
| 推荐 | TrueHUD、PrismaUI、Maxsu Poise、Modern Stagger Lock |
| 可选强化 | OCF，可带来更细的武器分类兼容 |

说明：

- 没有 TrueHUD 时，架势条与部分反馈会不完整。
- 没有 PrismaUI 时，战技菜单与 HUD 无法正常使用。
- 没有 Maxsu Poise 与 Modern Stagger Lock 时，当前版本的常规硬直体验会明显不完整。
- 没有 OCF 时，项目仍能运行，但武器分类会退回较粗的原版判断。

### 当前可用的用户入口

- 在背包菜单打开时，按 T 可以开关战技分配菜单。
- 默认按 Left Alt 可以切换战技状态。
- 当前战技菜单会跟随背包里选中的武器刷新。
- 当前仓库自带一组 Elden Rim 风格的样例战技，可直接用于测试系统链路。

### 当前可调配置

- 基础设置位于 Data/SKSE/Plugins/RimCombat/Settings.json。
- 默认值位于 Data/SKSE/Plugins/RimCombat/SettingsDefault.json。
- 战技定义位于 Data/SKSE/Plugins/RimCombat/WeaponArt/*.json。
- 本地化文本位于 Data/SKSE/Plugins/RimCombat/Localization.json。

如果你现在的目标是试玩与调参，那么最值得优先观察的，是下面这几项：

1. 不同武器类型下的耐力成本与格挡强度是否符合预期。
2. 力竭、架势、破防之间是否形成了清晰的攻防节奏。
3. 限时格挡是否足够有收益，但又不会过强。
4. 战技的开启、分配、法术效果与 HUD 提示是否顺畅。
5. 特殊硬直与外部韧性模组的衔接是否自然。

### 当前打包内容

仓库中的 dist 目录已经包含当前运行时资源，包括：

- SKSE 插件与配置文件
- 战技 JSON、设置文件与本地化文件
- PrismaUI 视图资源
- OAR 动画配置
- 当前打包归档

如果你只是想体验当前版本，优先使用 dist 中已经整理好的运行时内容，而不是自行从源码目录拼装。

## 面向玩家的结论

如果你想要的是一套已经完全收尾、内容量充足、演出稳定的正式战斗 overhaul，Rim Combat 现在还没到那个阶段。

但如果你想测试一套更强调耐力消耗、力竭惩罚、架势压制、格挡博弈、战技切换和特殊硬直联动的近战原型，那么它现在已经值得进游戏认真测试。
